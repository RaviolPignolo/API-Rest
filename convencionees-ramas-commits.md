# Ideas convenciones para el nombre de las ramas y posibles commits

## Rama main
Esta es la rama principal donde el código fuente refleja el estado de producción actual.

## Ramas de features nuevos
Estas ramas se usan para desarrollar nuevas características o cambios en el código. Suelen nombrarse de manera que reflejen la característica o el trabajo que se está realizando. Por ejemplo:

Endpoint/listarTarea
Endpoint/crearTarea
Endpoint/detalleTarea

## Ramas de Corrección de Errores
Para la corrección de errores, las ramas pueden seguir un patrón similar a las ramas de características pero destacando que están enfocadas en correcciones. Por ejemplo:

bugfix/listarTarea
bugfix/crearTarea
bugfix/detalleTarea

## Buenas Prácticas para ramas
Mantén los nombres de las ramas cortos pero descriptivos.
Usa guiones para separar palabras.
Evita caracteres especiales o espacios.


## convenciones mensajes commits
Limite de 50 caracteres
La primera letra en Mayus
No terminar el mensaje con punto final
Utilizar el modo imperativo

ejemplos
Se creo el controlador y la ruta...
Se desarrollo las funciones del controlador...
Se arreglo un problema con la ruta...
